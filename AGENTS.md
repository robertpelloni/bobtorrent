# Agents & AI Orchestration

This project utilizes a multi-agent architecture to achieve autonomous development.

## 📜 Universal Directives
> **All agents must strictly follow the Universal LLM Instructions globally:**
> [UNIVERSAL_LLM_INSTRUCTIONS.md](docs/UNIVERSAL_LLM_INSTRUCTIONS.md)

## 🤖 Active Agents

### 1. Director (The Architect)
-   **Role**: High-level planning, system design, and roadmap management.
-   **Responsibilities**:
    -   Maintaining `VISION.md`, `ROADMAP.md`, `MEMORY.md`, and the `TODO.md` backlog.
    -   Executing extreme-detail analysis on codebase history.
    -   Enforcing the Multi-Agent Mega Protocol.

### 2. Builder (The Engineer)
-   **Role**: Implementation and coding.
-   **Responsibilities**:
    -   Wiring up 100% of the features in UI and Backend.
    -   Writing extremely deep, analytical code comments.
    -   Executing intelligent, zero-loss Git merges and submodule updates.

### 3. QA (The Tester)
-   **Role**: Verification and Validation.
-   **Responsibilities**:
    -   Running exhaustive test suites.
    -   Incrementing global versions in `VERSION` and `CHANGELOG.md`.

## 🔄 Workflow
1.  **Sync**: Intelligently merge feature branches/upstream.
2.  **Analyze**: Produce accurate ROADMAP.md and TODO.md models.
3.  **Act**: Build the feature robustly and perfectly.
4.  **Document**: Create HANDOFF.md and update Dashboards.
5.  **Push**: Commit, Push, and loop autonomously.
